## What we build

**resist.js** is a local-first software company. We build applications that run
on your device, keep your data yours, and work whether or not the network does.
The monorepo stays private while products are in development ‚Äî expect it to
open up as they mature. In the meantime, this profile is where we say what we
stand for.

## How we work

- **Local-first** ‚Äî your data lives on your device, syncs on your terms, and
  survives being offline. The cloud is an accessory, not a landlord.
- **Privacy by default** ‚Äî nothing leaves your machine unless you ask it to. No
  telemetry-by-surprise, no quiet exfiltration.
- **Craft** ‚Äî small surface area, honest defaults, and software that ages well.
  We ship what we would want to run ourselves.

### [foundation-registry](https://github.com/kaelys-js/foundation-registry) `wip`

_Every decision the firm has made, stored as versioned Markdown records with schemas and clear ownership._

Firm-level decisions scatter across wikis, chat, and one-off docs, and nobody can find the reasoning six months later. This repo keeps them as Markdown-with-YAML-frontmatter records ‚Äî PRDs, PDRs, ADRs, SPECs, Tasks, Releases ‚Äî validated by JSON Schema and routed to named teams via CODEOWNERS. Records partition by domain (governance, design, frontend, backend, infra, security, data, observability) and cross-link forward: PRD ‚Üí PDR/ADR ‚Üí SPEC ‚Üí Tasks ‚Üí Release. It's the record of what the firm decided and why ‚Äî not a runtime application.

**Products**

- `config` ‚Äî Shared configs for every tool the firm uses ‚Äî tsconfig base, oxlint, oxfmt, taplo, syncpack, markdownlint, yamllint, commitlint, gitleaks, editorconfig, lefthook.
- `core` ‚Äî Small runtime helpers (repoRoot, miseExec) that qa and sync both depend on.
- `qa` ‚Äî Lint, format, and structural checks across the whole tree ‚Äî gitmeta, editorconfig, lefthook, and JSON-schema validation, all under one command.
- `sync` ‚Äî Drift guard for generated files ‚Äî version pins, vendored schemas, Turbo config, and the post-checkout/merge hooks that keep them in step.
- `registry` ‚Äî The record spine itself ‚Äî PRD, PDR, ADR, SPEC, tasks, releases ‚Äî validated against a JSON Schema and rendered as a browseable index site.
